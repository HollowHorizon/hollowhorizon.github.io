Hollow Engine Roadmap

От фундамента до совершенства

RELEASE 2.0

The Foundation

Базовый цикл разработки. Нажмите на пункты для подробностей.

  • Блоки кода Визуальное программирование ▼
    Понижает порог входа в движок. Временная альтернатива скриптам. Уже реализовано ~30% блоков из MCreator и десятки кастомных "suspend" блоков.
  • ECS (Geary) Ядро движка ▼
    Geary ECS с архетипами в основе. Разработка конвертера: ECS <-> KotlinX Serialization <-> Kool UI для удобного редактирования Data-классов без написания UI вручную.
  • Редактор NPC + Компоненты ▼
    Предпросмотр NPC, спавн и настройка их ECS компонентов через интерфейс. Базовое знакомство с Entity Component System.
  • Модели и анимации ▼
    Настройка видимых костей, шейп-кеев, костей для предметов. Система анимаций (State Machine) с графом и условиями перехода, аналогичная Unity.
RELEASE 2.1

Data-Driven & Scripting

Полный контроль через код и файлы. Самый масштабный этап.

  • Kotlin Scripting (.kts) + IDE ▼
    Встроенная IDE с подсветкой синтаксиса. Снимает ограничения блоков кода, даёт полный контроль над логикой. Поддержка ScriptDefinition.
  • Scripted UI Декларативный интерфейс ▼
    Создание интерфейсов через KTS скрипты. Позволяет скриптерам создавать свои HUD и меню до появления визуального редактора.
  • Asset Manager Внутриигровая IDE ▼
    Просмотр и редактирование JSON, .mcfunction, картинок и моделей. Удобная работа с префабами и ресурсами без сворачивания игры.
  • Система аддонов ▼
    Изоляция контента. Аддоны можно включать/отключать без перезапуска. Просто распространять и делиться контентом.
  • Events 2.0 Оптимизация ▼
    Оптимизация текущей системы событий, повышение эффективности, особенно на клиентской стороне.
RELEASE 2.2

Storytelling

Инструменты для режиссуры, кино и атмосферы.

  • Катсцены и Таймлайн ▼
    Секвенсор для настройки камеры, анимаций NPC и мира. Интеграция с физическим движком как "киллер-фича".
  • Cinematic Audio ▼
    Независимая аудио-система для катсцен. Звуки "запекаются" под сцену для идеальной синхронизации, в отличие от динамического геймплейного аудио.
  • Шейдеры и Постпроцессинг ▼
    Виньетка, ЧБ-фильтр, сепия, хроматическая аберрация, импакт-фреймы для боевых сцен, размытие и фокус в катсценах.
  • Процедурные анимации ▼
    Рэгдоллы, физика одежды и волос. Добавляет реализма в катсцены и взаимодействие с миром.
RELEASE 2.3

Gameplay Loop

RPG механики, NPC и взаимодействие.

  • Редактор диалогов (Graph) ▼
    Визуальный редактор диалогов на основе графов. Интеграция с UI для отображения выборов и реплик.
  • Продвинутый Pathfinder ▼
    NPC учатся паркурить и не застревать. Исправление проблем ванильного навигатора (кручение на месте).
  • Компаньоны и Социум ▼
    Охота, помощь в добыче, защита. Системы репутации (локальная/фракционная). Настройка рынков, цен, реакций и даже любовных линий.
  • NPC Боссы ▼
    Скриптуемые боссы с фазами, паттернами атак, анимациями и кастомными босс-барами.
  • Visual UI Editor ▼
    Визуальный редактор поверх скриптовой системы UI. Drag-and-drop виджетов для простых меню.
RELEASE 2.4

Modpack Tools

Инструментарий для создания полноценных игр/модпаков.

  • Runtime Registry (Сущности) ▼
    Создание новых типов блоков/предметов/сущностей через файлы (Префабы) без хардкода. Обход ванильных регистров.
  • Система Структур ▼
    Точка входа для модпаков. Генерация мира, хранение информации о NPC, триггерах и катсценах внутри структур. Достаточно просто интергрировать со скриптами
  • Глобальное Аудио ▼
    Процедурное аудио, стриминг, многослойность на основе условий (расстояние, фаза боя). Привязывается к структурам или измерению.
  • Редактор рецептов ▼
    Аналог KubeJS, но через UI. Фильтрация, поиск, просмотр изменений. Поддержка новых типов рецептов через скрипты.
    • Примечание: Многое из этого вероятно хотелось бы добавить и раньше, но оно жёстко привязано к структурам, а это долгая и сложная система, которая к тому же постоянно переписывается моджангами.
RELEASE 2.5

Visuals & Polish

Графика нового поколения и оптимизация.

  • GPU Particles ▼
    Система частиц на основе вычислительных шейдеров (Compute Shaders). Аналог Unity Particle System / Photon.
  • Instancing (GLTF) ▼
    Поддержка инстансинга для кастомных моделей движка. Повышение производительности рендера в десятки раз.
  • Advanced Lighting ▼
    (Экспериментально) Deferred Shading, Volumetric Fog. Требует сложного вмешательства в пайплайн шейдеров.
  • OptiFine Support ▼
    Если будет спрос, то можно было бы написать и для него интеграцию, хоть и сидеть с дебаггером особо не хочется.
FUTURE CONCEPTS

Вне планов

Эти пункты могут быть добавлены в любой момент или вообще заменены, так что воспринимайте скорее как идеи, чем планы.

  • Quest System (Graph) ▼
    Квестовая система на основе графов в духе FTB Quests, но с возможностью прикреплять систему квестов к определённому нпс или блоку, предмету, а не только игроку. Также можно сделать систему довольно гибкой, чтобы пользователи могли даже делать свои системы прокачки.
  • Dev Tools ▼
    Интеграция ботов, мониторинга, профайлинга, команд и администрирования. Возможно встроенная поддержка Git.
